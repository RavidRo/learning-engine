## Users

Signal Garden is for technical users who want one app for organizing news and
updates around their interests. The current primary user is Ravid, using it to
stay current on selected people, organizations, technologies, and subjects.
Future users should be able to use the same model without needing the app to run
only on a local machine.

AI agents such as Hermes are also downstream users of the product context. They
read curated interests and sources through an MCP integration so they can
prepare useful briefings from the same organized signal graph.

## Product Purpose

Signal Garden helps users define interests, attach sources, review fresh
updates, and save useful items into collections. It exists to turn scattered
information streams into a maintainable source of timely, interest-specific
signals.

Success means a technical user can quickly see what changed across the topics
they care about, understand the source of each update, collect important items,
and give AI agents enough structured context to prepare higher-quality
briefings.

## Brand Personality

Signal Garden should feel calm, precise, and trustworthy. The product voice is
direct and work-focused: it should help users scan, compare, and act without
ornamental noise.

The garden metaphor should remain organized and useful, not cute, rustic, or
decorative. AI-assisted moments should explain outcomes and next actions plainly.

## Anti-references

Do not make the app feel like a marketing landing page, a social feed optimized
for engagement, or a decorative garden-themed interface. Avoid large green
gradients, leaf-pattern backgrounds, playful botanical illustrations, nested
cards, and dashboard chrome that competes with the updates themselves.

Avoid hiding failures behind quiet fallbacks. When sources cannot be collected,
collections cannot load, or agent integrations are unavailable, the UI should
show clear product states and recovery paths.

## Design Principles

Clear signal over decoration: every surface should improve scanning, comparison,
or action on interests, sources, updates, and collections.

Operational calm: use restrained hierarchy, stable controls, and consistent
states so the app can support repeated daily use.

Explain the system: source status, collection actions, freshness windows, and AI
agent integration points should be legible without requiring implementation
knowledge.

Local clarity, future multi-user fit: preserve the current personal workflow
while designing labels, empty states, and permissions-sensitive moments so they
can scale beyond one local user.

Agent-ready structure: interests and sources should remain explicit product
concepts that humans can manage and AI agents can consume reliably.

## Accessibility & Inclusion

Use the usual product baseline: keyboard usability, visible focus states,
reduced-motion support, readable contrast, and responsive layouts for phone and
desktop. No additional project-specific accessibility target has been set.
